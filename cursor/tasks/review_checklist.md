# Review Checklist

## Scope and alignment
- [ ] Changes match the stated goals and constraints.
- [ ] No unrelated refactors or scope creep.

## Correctness and safety
- [ ] Logic is correct for primary and edge cases.
- [ ] Guardrails are respected (no internet, no secrets, no destructive ops).
- [ ] Data handling and privacy constraints are respected.

## Quality and maintainability
- [ ] Changes are minimal and localized.
- [ ] Documentation is updated where needed.

## Testing and execution
- [ ] Tests or commands needed are listed.
- [ ] If no tests were run, the risk is documented.

## Review output
- [ ] Findings ordered by severity with file and line references.
- [ ] Residual risks or gaps stated explicitly.
